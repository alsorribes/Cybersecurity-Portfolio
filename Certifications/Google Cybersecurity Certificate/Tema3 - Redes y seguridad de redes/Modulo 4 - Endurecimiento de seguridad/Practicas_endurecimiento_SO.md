# Prácticas de endurecimiento del SO

El sistema operativo es la interfaz entre el hardware y el usuario y es el primer programa que se carga cuando se enciende el ordenador. Actúa como intermediario entre las aplicaciones de software y el hardware del ordenador. Es muy importante tener un SO seguro ya que de no ser así puede llegar a comprometer toda una red. 

Algunas de las tareas de endurecimiento que podemos encontrar en intervalos regulares son:

- **Instalación de parches:** Una actualización de parches es una actualización de software y SO que aborda vulnerabilidades de seguridad dentro de un producto. Es importante hacer las actualizaciones cuanto antes ya que al publicar el parche, los atacantes pueden saber dónde está exactamente la vulnerabilidad y atacarla.

- **Imagen de línea base:** Conjunto documentado de especificaciones dentro de un sistema que se utiliza como base para futuras construcciones y actualizaciones. 

- **Eliminación de hardware y software:** Garantiza que todo el hardware antiguo y el software obsoleto se eliminen adecuadamente ya que pueden contener vulnerabilidades. 

- **Implementación de políticas de contraseñas segura:** Estas políticas requieren que las contraseñas sigan unas reglas concretas. Para evitar a las personas con malas intenciones, una política suele establecer que un usuario perderá el acceso a la red tras introducir una contraseña incorrecta un determinado número de veces, por ejemplo.
