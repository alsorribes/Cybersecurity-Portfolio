# Administrador de paquetes para instalar aplicaciones

### Introducción a los administradores de paquetes

Un paquete es una pieza de software que puede combinarse con otros paquetes para formar una aplicación. Estos paquetes contienen los archivos necesarios para instalar una aplicación ya que estos incluyen dependencias, que son archivos suplementarios utilizados para ejecutar una aplicación.

Los administradores de paquetes pueden ayudar a resolver cualquier problema con las dependencias y realizar otras tareas de gestión. Es importante tener siempre la versión más reciente instalada.
<br>

### Tipos de administradores de paquetes

Muchas distribuciones de Linux de uso común derivan de la misma distribución madre. Esto es útil a la hora de instalar aplicaciones porque determinados administradores de paquetes funcionan con determinadas distribuciones. 

Los distintos administradores de paquetes suelen utilizar diferentes extensiones de archivo. Por ejemplo, el administrador de paquetes de Red Hat usa la extensión .rpm y Debian y sus derviados usan la extensión .deb.
<br>

### Herramientas de gestión de paquetes

También existen herramientas de gestión de paquetes que le permiten trabajar fácilmente con los paquetes a través del shell. Estas herramientas se usan a veces en lugar de los administradores de paquetes porque permiten a los usuarios realizar más fácilmente tareas básicas. Dos herramientas destacables son:

- **Herramienta avanzada de paquetes (APT):** APT es una herramienta que se utiliza con las distribuciones derviadas de Debian. Se ejecuta desde el CLI para gestionar, buscar e instalar paquetes.

- **Yellowdog Updater Modificado (YUM):** YUM es una herramienta utilizada con las distribuciones derivadas de Red Hat. Se ejecuta desde la CLI para gestionar, buscar e instalar paquetes.
