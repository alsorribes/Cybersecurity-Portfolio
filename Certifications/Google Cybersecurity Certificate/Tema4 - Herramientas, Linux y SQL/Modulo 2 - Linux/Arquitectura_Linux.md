# Arquitectura de Linux

Linux está formado por varios componentes que son los siguientes:

- **Usuario:** Es la persona que interactúa con el dispositivo y que inicia las tareas o los comandos que va a ejecutar el SO. Linux es un sistema multiusuario lo que significa que más de un usuario puede usar los recursos del sistema a la vez.

- **Aplicaciones:** Es un programa que realiza una tarea específica. Las aplicaciones de Linux se distribuyen normalmente a través de administradores de paquetes. Un administrador de paquetes es una herramienta que ayuda a los usuarios a instalar, gestionar y eliminar paquetes o aplicaciones. Un paquete es una pieza de software que puede combinarse con otros paquetes para formar una aplicación. 

- **Shell:** Es la forma en que nos comunicaremos con el sistema ya que es un intérprete de comandos. Su función es procesar los comandos y generar resultados. 

- **Estándar de jerarquía del sistema de archivos (FHS):** Es el componente que organiza los datos para poder encontrarlos cuando el sistema acceda a ellos. Define cómo se organizan los directorios, su contenido y otros tipos de almacenamiento para que el SO sepa dónde encontrar datos concretos.

- **Kernel:** Es el componente que administra los procesos y la memoria del SO. Se comunica con el hardware para ejecutar los comandos enviados por el shell y usa controladores para permitir que las aplicaciones ejecuten tareas.

- **Hardware:** Son los componentes físicos del dispositivo.
