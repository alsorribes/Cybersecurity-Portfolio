# Marcos y controles


### Marcos (frameworks)

Los marcos de seguridad (o frameworks) son directrices utilizadas para construir planes que ayuden a mitigar los riesgos y amenazas a los datos y la privacidad. La seguridad no es solo virtual sino que también física como por ejemplo el acceso a la oficina que puede requerir el uso de una tarjeta de identificación.

Otros marcos de seguridad proporcionan información sobre cómo prevenir, detectar y responder a las brechas de seguridad. Esto es importante cuando se trata de proteger una organización de ataques de ingeniería social ya que las personas son la mayor amenaza para la seguridad. Por ello, se usan marcos para crear planes que aumenten la concienciación de los empleados.


### Controles

Los controles de seguridad son defensas diseñadas para reducir riesgos de seguridad específicos. Hay tres tipos de controles a destacar:

- **Encriptación:** Es el proceso de convertir los datos de un formato legible a un formato codificado, que será ilegible para humanos y ordenadores. Este tipo de controles se usan para proteger los datos sensibles de la organización.

- **Autenticación:** Es el proceso de verificar quién es alguien o algo. Un ejemplo sería el de registrarse en un sitio web con usuario y contraseña. Hay métodos más avanzados cómo el uso de datos biométricos, que son las características físicas únicas de una persona, para verificar su identidad.

- **Autorización:** Es el concepto de otorgar acceso a recursos específicos dentro de un sistema a una persona.
