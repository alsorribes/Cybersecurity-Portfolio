# Dominios de seguridad del CISSP

El CISSP identifica ocho dominios diferentes. Los equipos de seguridad los usan para organizar sus tareas diarias e identificar las brechas que puede haber en la seguridad que podrían provocar consecuencias negativas para una organización. 


### 1- Seguridad y gestión de riesgos:

Este dominio se subdivide en varias áreas de interés, que son las siguientes:

- **Definición de metas y objetivos de seguridad:** Al definir metas y objetivos de seguridad las organizaciones pueden reducir los riesgos para los recursos y datos críticos como la PII.

- **Mitigación de riesgos:** Disponer de los procedimientos y normas adecuados para reducir rápidamente el impacto de un riesgo.

- **Cumplimiento normativo:** Método principal utilizado para desarrollar las políticas de seguridad internas de una organización, los requisitos y normas.

- **Continuidad del negocio y regulaciones legales:** Capacidad de una organización para mantener su productividad diaria mediante el establecimiento de planes de recuperación ante desastres por riesgos y seguir las normas de comportamiento ético para minimizar la negligencia.


### 2- Seguridad de los activos:

Este dominio se centra en proteger los activos físicos y digitales. También se relacionan con él el almacenamiento, mantenimiento, retención y destrucción de datos ya sea del tipo que sean. Además, las organizaciones deben tener políticas y procedimientos que garanticen que los datos cumplen con los requisitos anteriores ya que, saber que datos sé tienen y quién tiene acceso a ellos, es esencial para tener una postura de seguridad sólida. 


### 3- Arquitectura de seguridad y la ingeniería: 

Se centra en optimizar la seguridad de los datos garantizando que existen herramientas, sistemas y procesos eficaces para proteger los recursos y los datos de una organización. Hay un concepto importante a destacar:

- **Responsabilidad compartida:** Todas las personas de una organización asumen un papel activo en la reducción del riesgo y en el mantenimiento de la seguridad tanto física como digital. 


### 4- Seguridad de las comunicaciones y las redes:

Gestiona y protege las redes físicas y las comunicaciones inalámbricas. Una red segura mantiene a salvo los datos y las comunicaciones de una organización ya se presencial, en la nube o conectandose a servidores de forma remota. 


### 5- Gestión de identidad y acceso (IAM):

Este dominio se centra en el acceso y la autorización para mantener los datos seguros asegurándose de que los usuarios siguen las políticas establecidas para controlar y gestionar los activos. Su objetivo es reducir el riesgo global para los sistemas y los datos. Hay 4 componentes a tener en cuenta:

- **Identificación:** Un usuario verifica quién es proporcionando un nombre de usuario, una tarjeta de acceso o datos biométricos.

- **Autenticación:** Proceso de verificación para probar la identidad de una persona, como introducir una contraseña o un PIN. 

- **Autorización:** Tiene lugar una vez que se ha confirmado la identidad del usuario y se relaciona con su nivel de acceso, que dependrá de su función en la organización en la que pertenezca. 

- **Responsabilidad:** És la supervisión y el registro de las acciones del usuario, como los intentos de inicio de sesión, para demostrar que los sistemas y los datos se utilizan correctamente.


### 6- Evaluación y pruebas de seguridad

Dominio que se basa en la realización de pruebas de control de la seguridad, la recopilación y el análisis de datos y la realización de auditorías de seguridad para supervisar los riesgos, las amenazas y las vulnerabilidades. Todo esto puede ayudar a una organización a identificar nuevas y mejores formas de mitigar dichas amenazas. 

Un analista puede utilizar estas evaluaciones y los informes de la evaluación para mejorar los controles existentes o implementar nuevos controles. 


### 7- Operaciones de seguridad

Se centra en realizar investigaciones e implementar medidas preventivas cuando se detecta un incidente de seguridad. Si hay un ataque activo, mitigar el ataque y evitar que siga escalando es esencial para garantizar que la información privada esté protegida de los agentes de amenaza. Cuando se neutraliza, se recopilan pruebas digitales y físicas para hacer una investigación forense. De esta manera se va a poder identificar cuándo, cómo y por qué se produjo la brecha y así poder aplicar medidas preventivas.


### 8- Seguridad en el desarrollo de software

Es el uso de práticas de programación seguras. Durante el ciclo de vida de desarrollo de software, un paso importante es el de la seguridad. Al garantizar que cada fase del ciclo de vida de desarrollo de software se somete a revisiones de seguridad, la seguridad puede integrarse completamente en el producto final. Esto mantiene el software y los datos sensibles protegidos.
